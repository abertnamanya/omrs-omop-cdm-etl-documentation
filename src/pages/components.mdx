# Using Components

The defining feature of MDX is the ability to import and use React components
directly in your content.

## Local imports

```mdx
import { Callout } from '../../components/Callout'

## Installation

<Callout type="tip">
  Always pin your MDX version in production!
</Callout>
```

## MDXProvider — global components

```jsx
import { MDXProvider } from '@mdx-js/react'

const components = {
  h1: (props) => <h1 className="heading" {...props} />,
  code: SyntaxHighlighter,
  Callout,
}

<MDXProvider components={components}>
  <App />
</MDXProvider>
```

## Best practices

1. Create a shared components barrel file for easy importing
2. Use MDXProvider for site-wide components like Callout, Note, Badge
3. Keep component imports at the top — never inline
4. Test components in isolation before embedding in MDX
